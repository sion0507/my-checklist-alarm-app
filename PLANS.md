# PLANS.md

이 문서는 planner가 실제 구현 계획을 세우기 위한 제품/기능 기준이다. 여기서는 구현 코드를 작성하지 않는다.

## Planner-ready task specification

`my-checklist-alarm-app`은 개인용 iPhone PWA 체크리스트 알림 앱이다. 사용자는 할 일을 날짜/시간/반복 규칙과 함께 저장한다. 앱은 설정된 아침 시간에 오늘 할 일을 정리해 알려주고 추가 입력을 유도한다. 시간이 지정된 할 일은 해당 시간에 확인 알림을 보낸다. 매일 기본 23:00에는 미완료 할 일을 보여주고, 사용자가 각 항목을 삭제하거나 다른 날짜로 넘길 수 있게 한다. 데이터 저장은 MVP에서 로컬 저장을 사용한다. 가장 중요한 요구사항은 iPhone 잠금화면에 도착하는 실제 푸시 알림이며, 이를 위해 PWA/Web Push/iOS 제약을 검토하고 가능한 구현 경로를 계획해야 한다.

## Recommended implementation phases

### Phase 0: Feasibility spike for iPhone PWA notifications

목표: 핵심 목표인 실제 푸시 알림 가능성을 먼저 검증한다.

Planner가 확인해야 할 것:

- iOS PWA Web Push 요구 조건
- 홈 화면 설치 필요 여부
- Service Worker/Push API 지원 범위
- 로컬 예약 알림만으로 가능한 범위
- 푸시 예약 발송에 백엔드가 필요한지
- 알림음/노래 커스터마이즈 가능 여부

Acceptance criteria:

- iPhone PWA에서 가능한 알림 방식이 문서화됨
- “진짜 푸시 알림” 구현 경로가 결정됨
- 노래/알림음 설정 가능 여부가 결정됨
- 불가능한 부분은 MVP 제외/후속 과제로 명시됨

### Phase 1: Core local checklist MVP

기능:

- 오늘 화면
- 월간 캘린더 화면
- 연/월 선택 UI
- 각 날짜 칸에 해당 날짜 할 일 요약 표시
- 좌우 슬라이드로 이전/다음 월 이동
- 할 일 목록
- 빠른 추가
- 할 일 추가/수정/삭제
- 완료 체크
- 날짜 지정
- 선택적 시간 지정
- 로컬 저장

Acceptance criteria:

- 새 할 일을 추가하면 새로고침 후에도 남아 있음
- 오늘 날짜의 할 일이 오늘 화면에 표시됨
- 캘린더에서 선택한 연/월의 날짜 칸마다 해당 날짜 할 일이 표시됨
- 캘린더를 좌우로 슬라이드하면 이전/다음 월로 이동함
- 완료 체크가 저장됨
- 할 일을 수정/삭제할 수 있음

### Phase 2: Settings and scheduling model

기능:

- 아침 알림 시간 설정
- 저녁 리마인드 시간 설정, 기본 23:00
- 반복 규칙 설정: 없음/매일/매주/매월
- 알림 권한 상태 표시
- 테스트 알림 버튼

Acceptance criteria:

- 아침/저녁 시간이 저장되고 변경 가능함
- 반복 할 일이 오늘 화면에 올바르게 표시됨
- 테스트 알림이 가능한 환경에서 동작함

### Phase 3: Morning check-in workflow

기능:

- 설정된 아침 시간 알림
- 오늘 할 일 요약
- 앱 내 아침 체크인 카드
- “추가로 할 일 있어?” 입력 유도

Acceptance criteria:

- 아침 체크인 카드가 오늘 첫 진입 시 표시됨
- 오늘 할 일 개수/목록이 요약됨
- 빠른 추가 입력이 가능함

### Phase 4: Time-specific reminders

기능:

- 시간이 지정된 할 일에 대해 해당 시간 알림
- 알림 클릭/앱 진입 시 해당 할 일 강조
- 완료/미루기 동작 제공

Acceptance criteria:

- 시간 지정 할 일이 알림 대상으로 등록됨
- 해당 시간에 알림이 발송됨
- 앱에서 해당 항목을 쉽게 완료 처리할 수 있음

### Phase 5: 23:00 unfinished-task review

기능:

- 기본 23:00 미완료 알림
- 미완료 목록 화면
- 항목별 삭제
- 항목별 다른 날짜로 넘기기

Acceptance criteria:

- 미완료 할 일이 있을 때 저녁 알림이 발송됨
- 미완료 목록에서 삭제 가능함
- 다른 날짜로 넘기면 날짜가 변경되고 오늘 목록에서 빠짐

## Calendar UI requirements from reference image

월간 캘린더는 사용자가 제공한 iPhone 캘린더 예시 이미지를 기준으로 계획한다.

Layout:

- iPhone 세로 화면 우선 디자인.
- 상단 좌측에 큰 월 제목. 예: `5월`.
- 상단 우측에 둥근 pill 액션 영역. MVP에서는 더보기/설정/주요 액션 버튼을 배치할 수 있다.
- 월 제목 아래에 넓은 둥근 카드 영역. MVP에서는 선택 날짜 요약, 오늘 요약, 아침 체크인, 저녁 미완료 정리 카드 중 하나를 표시하는 영역으로 사용한다.
- 그 아래에 7열 월간 캘린더 그리드.
- 요일 헤더는 `일 월 화 수 목 금 토`.
- 하단에는 floating rounded tab bar를 둔다. 기본 탭 후보는 `오늘`, `캘린더`, `설정/내 정보`이며 선택 탭은 회색 pill 배경으로 강조한다.

Calendar cell behavior:

- 각 날짜 칸에는 날짜 숫자와 해당 날짜의 할 일 pill을 표시한다.
- 일요일/공휴일은 빨간색, 토요일은 파란색, 평일은 검은색/짙은 회색으로 표시한다.
- 이전/다음 월 날짜는 낮은 opacity로 표시한다.
- 오늘 날짜는 파란 원형 배경으로 강조한다.
- 날짜를 탭하면 해당 날짜의 할 일 목록/추가 화면을 연다.
- 좌우 스와이프는 이전/다음 월 이동으로 동작한다.
- 연/월 선택 UI로 특정 년도/월에 직접 이동할 수 있다.

Todo pill behavior:

- 기본 스타일은 연한 파란색 배경 + 왼쪽 파란 강조선 + 회청색 텍스트.
- 시간이 있는 할 일은 `12시 직거`, `18시 꽃`처럼 시간 prefix를 표시할 수 있다.
- 시간이 없는 할 일은 제목만 표시한다.
- 긴 제목은 한 줄 ellipsis 처리한다.
- 한 날짜에 할 일이 많으면 표시 가능한 개수까지만 보여주고 `+N` 또는 더보기 표시를 둔다.
- pill을 탭하면 해당 할 일 상세/수정으로 이동한다.

Acceptance criteria additions:

- 캘린더 화면이 참조 이미지와 유사한 큰 월 제목, 넓은 7열 그리드, 날짜별 pill 표시, floating bottom tab bar 구조를 가진다.
- 좌우 스와이프로 월 이동이 가능하다.
- 특정 날짜를 탭해 해당 날짜의 할 일 목록을 확인하고 새 할 일을 추가할 수 있다.
- 오늘 날짜, 주말, 공휴일/기념일, 이전/다음 월 날짜가 시각적으로 구분된다.

## Navigation and UX decision log

- 앱의 기본 진입 화면은 `오늘` 화면으로 한다.
- 월간 캘린더는 하단 탭의 `캘린더` 탭에서 접근한다.
- 이유: 앱을 열자마자 “오늘 해야 할 일”과 아침/저녁 체크인 카드가 바로 보여야 체크리스트 알림 앱의 핵심 목적에 맞는다.
- 캘린더는 한 달 할 일 분포를 확인하고 특정 날짜의 할 일을 관리하는 보조 핵심 화면으로 둔다.
- 하단 탭 기본 구성은 `오늘`, `캘린더`, `설정`으로 시작한다.
- 할 일 추가 UX는 상단 빠른 입력 + 상세 모달을 기본으로 한다.
- 오늘 화면 상단 빠른 입력은 제목만으로 오늘 할 일을 즉시 추가한다.
- 날짜, 시간, 반복, 메모 등 추가 필드는 상세 모달에서 설정한다.
- 캘린더에서는 날짜를 탭한 뒤 해당 날짜를 기본값으로 상세 추가 모달을 연다.
- 오늘 화면은 완료/미완료 할 일을 모두 표시한다.
- 미완료 할 일은 상단에 먼저 표시한다.
- 완료된 할 일은 체크/흐린 스타일로 하단에 표시한다.
- 알림을 눌러 앱에 들어오면 관련 할 일을 강조한다.
- 알림 진입 상세/강조 상태에서는 빠른 액션 `완료`, `삭제`, `다른 날짜로 넘기기`를 제공한다.
- 반복 할 일에서 완료/삭제/날짜 변경의 기본 동작은 해당 날짜 발생분에만 적용한다.
- 반복 규칙 전체 수정/삭제는 상세 화면의 별도 액션으로 제공한다.
- 할 일 상세 모달 MVP 필드는 제목, 날짜, 시간, 반복, 메모, 알림 on/off를 포함한다.
- 아침 체크인 알림 기본 시간은 08:00이며 설정에서 변경 가능하다.
- 아침 체크인 카드는 알림 수신 여부와 관계없이 매일 오늘 화면에 표시한다.
- 사용자가 당일 `확인 완료`하면 그날은 접히거나 숨김 처리한다.
- 밤 미완료 정리 알림은 미완료 할 일이 있을 때만 발송한다.
- 앱 안의 저녁 정리 카드는 매일 표시하되, 미완료가 없으면 `오늘 할 일 모두 완료` 같은 완료 메시지를 보여준다.
- 수동 백업/복원은 MVP에서 제외한다.
- 로컬 저장 데이터 삭제 리스크를 명시하고, JSON 내보내기/가져오기는 후속 기능으로 남긴다.
- 테스트 범위는 단위 테스트, E2E 테스트, 수동 iPhone PWA 체크리스트를 모두 포함한다.

## Confirmed product/architecture decisions

1. 실제 iPhone 잠금화면 푸시를 위해 최소 백엔드/스케줄러 사용을 허용한다.
2. 백엔드/스케줄러는 무료 클라우드 기준으로 계획한다. 유료 전환은 MVP 범위에서 금지한다.
3. 노트북 서버 상시 실행 방식은 사용하지 않는다.
4. 무료 티어 한계로 알림 지연/누락 가능성을 리스크로 명시한다.
5. 서버에는 푸시 발송에 필요한 최소 데이터만 저장한다: 알림 id, 발송 시각, 제목/짧은 문구, 알림 타입, 완료/취소 상태.
6. 로컬 IndexedDB/localStorage 데이터가 사용자 할 일의 원본이다.
7. 반복 할 일은 원본 반복 규칙 + 날짜별 예외 저장 방식으로 관리한다.
8. 할 일 변경 시 서버 예약을 즉시 동기화하고 앱 시작 시 향후 예약을 재동기화한다.
9. 서버 예약 범위는 향후 7일이다.
10. MVP 알림 범위는 아침 오늘 할 일 알림, 시간 지정 할 일 알림, 밤 23:00 미완료 정리 알림을 모두 포함한다.
11. 알림음/노래 설정은 Phase 0에서 가능성을 검증하고, 불가능하면 MVP 제외/제한 표시한다.
12. 설정값은 localStorage, 할 일/반복 예외/예약 상태는 IndexedDB를 사용한다.
